# VapeSpot — 100 Location Pages

## Fichiers dans ce dossier
- `locations.json` → Les données des 100 VapeSpot (slug, nom, adresse, coordonnées GPS, description, horaires)

## Prompt à copier/coller dans Replit

```
Tu as un fichier locations.json avec 100 entrées VapeSpot. Chaque entrée ressemble à ceci :

{
  "slug": "vapespot-sydney-cbd",
  "business_name": "VapeSpot Sydney CBD",
  "suburb": "Sydney CBD",
  "state": "NSW",
  "city": "Sydney",
  "address": "117 Clarence St, Sydney NSW 2000, Australia",
  "lat": -33.8670946,
  "lng": 151.2048423,
  "description": "Looking for a vape in Sydney CBD? VapeSpot stocks IGET, Alibarbar, HQD, Relx & Vaporesso. Open 24/7.",
  "hours": {
    "monday": "24 hours", "tuesday": "24 hours", "wednesday": "24 hours",
    "thursday": "24 hours", "friday": "24 hours", "saturday": "24 hours", "sunday": "24 hours"
  }
}

Tu dois générer 100 pages HTML individuelles, une pour chaque VapeSpot. 
Chaque page doit être un fichier .html nommé comme le slug (ex: vapespot-sydney-cbd.html).

Pour chaque page, inclus :
1. Le nom du VapeSpot en titre (h1)
2. L'adresse complète
3. Une carte Google Maps centrée sur les coordonnées (lat, lng) - utiliser une iframe Google Maps
4. La description
5. Les horaires (24/7)
6. Un bouton "Get Directions" qui ouvre Google Maps itinéraire
7. Design propre, responsive, couleurs sobres (noir/blanc/gris)
8. Un lien retour vers la page d'accueil ou une liste

Génère les 100 fichiers HTML dans un dossier /pages/.
```

## Après sur Replit
1. Crée un nouveau Replit (HTML/CSS/JS ou Static)
2. Colle le prompt ci-dessus
3. Upload `locations.json` dans le projet
4. L'IA va générer les 100 pages
5. Télécharge le dossier `/pages/` généré
6. Push sur GitHub dans ton repo vapespot.store

## Alternative plus rapide
Si Replit galère avec 100 fichiers d'un coup, dis-lui de générer un seul fichier `index.html` 
qui charge locations.json en JavaScript et affiche chaque page dynamiquement avec un routing basique.
C'est plus simple et plus propre.
